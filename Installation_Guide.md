# 🚀 홍다 Biz (Hongda Biz) 통합 설치 및 운영 가이드 (v4.1)

홍다 Biz는 매입/매출 거래명세서 관리, 재고 추적, 그리고 경매 데이터 자동 수집을 통합한 비즈니스 솔루션입니다.

---

## 💻 시스템 권장 사양

프로그램의 원활한 구동, 특히 여러 경매 사이트의 데이터를 동시에 수집할 때를 대비한 권장 사양입니다.

| 항목 | 최소 사양 | **권장 사양 (추천)** |
| :--- | :--- | :--- |
| **운영체제** | Windows 10 (64비트) | Windows 11 (64비트) |
| **CPU** | i3 (Dual Core) 이상 | **i5 (Quad Core) 이상** |
| **메모리(RAM)** | 4GB | **8GB 이상** |
| **저장장치** | SSD (여유 2GB) | **NVMe SSD 권장** |

## 📋 1. 사전 준비: 필수 설치 프로그램 (먼저 설치)

시스템 가동을 위해 아래 두 프로그램은 **반드시** 설치되어 있어야 합니다. 설치되지 않은 경우 다음 단계의 자동 설치가 진행되지 않습니다.

1.  **Node.js (v20 이상 LTS)**: [공식 다운로드 페이지](https://nodejs.org/)
2.  **MySQL Server (8.0 또는 8.4 LTS)**: [다운로드](https://dev.mysql.com/downloads/mysql/) 설치 시 설정한 루트 비밀번호를 반드시 기억하세요.

---

## ⚡ 2. 퀵 스타트: 원스톱 자동 설치 (강력 권장)

기초 프로그램이 설치되었다면, 아래 파일을 통해 **클릭 한 번**으로 모든 환경을 구축할 수 있습니다.

1.  **`Initial_Setup.bat` 실행**: 프로젝트 루트 폴더에 있는 이 파일을 더블 클릭합니다.
2.  **설치 진행**: 
    - 자동으로 `npm install`을 수행하여 라이브러리를 설치합니다.
    - MySQL 데이터베이스(`trade_management`)를 자동으로 구축합니다.
    - 경매 크롤링용 전용 브라우저 엔진을 설치합니다.
3.  **관리자 계정 초기화**: 마지막에 관리자 로그인 정보가 `admin / admin1234`로 자동 초기화됩니다.
4.  **완료**: 5~10분 후 화면에 완료 메시지가 뜨면 모든 준비가 끝납니다!

## 🔑 3. 기기 등록 및 라이선스 활성화 (필수)

본 프로그램은 보안을 위해 **기기 인증제**로 운영됩니다. 설치 후 반드시 아래 절차를 거쳐야 합니다.

1.  **런처 실행**: `hongda-biz-launcher/dist/HongdaBiz-win32-x64` 폴더의 **`HongdaBiz.exe`**를 실행합니다.
2.  **기기 ID 확인**: 런처 왼쪽 하단 **[기기 라이선스 정보]** 영역에서 내 기기의 ID(예: `HDB-ABC1234`)를 확인합니다.
3.  **등록 요청**: 해당 ID를 클릭하여 복사한 뒤, 관리자(사장님)에게 전달하여 등록을 요청하세요.
4.  **승인 대기**: 관리자가 등록을 완료하면, 런처 재실행 시 **[승인 완료]** 상태로 변하며 시스템 사용이 가능해집니다.

## 🚀 4. 시스템 운영 및 업데이트

### 서버 가동 (Zero-Wait Access)
- 런처 실행 시 라이선스가 유효하면 **서버가 자동으로 즉시 가동**됩니다.
- 브라우저 창이 뜨면 `admin / admin1234`로 로그인하여 즉시 업무를 시작하세요.

### 원클릭 온라인 업데이트
- 프로그램 개선 버전이 출시되면 런처 상단에 **"🚀 신규 업데이트 발견!"** 배너가 뜹니다.
- **[지금 업데이트]** 버튼을 클릭하면 원클릭으로 코드와 DB가 최신으로 패치됩니다.

---

## 🛠 5. 주요 유지보수 도구

- **`Initial_Setup.bat`**: 전체 재설치나 환경 복구가 필요할 때 사용 (런처 자동 종료 포함)
- **`Create_Distribution.bat`**: 런타임 데이터가 제외된 깨끗한 배포 패키지(ZIP) 제작 시 사용
- **`Update_System.bat`**: 수동 최신 버전 체크 및 패치 시 사용
- **비밀번호 분실 시**: `backend` 폴더에서 `node scripts/emergency_reset_admin.js` 실행 (admin1234로 초기화)
